# Service Request Overview

## Purpose

This document explains what a Service Request is and how it differs from an Incident.

---

## Definition

A Service Request is a formal request from a user for a standard IT service, information, access, or equipment.

Unlike incidents, service requests do not involve unexpected service interruptions.

---

## Examples

- New laptop
- Software installation
- VPN access
- Email account creation
- Shared folder access
- Printer installation
- Password change
- Monitor request

---

## Service Request Lifecycle

1. User submits request.
2. Service Desk reviews the request.
3. Required approvals are obtained.
4. Request is assigned.
5. Request is fulfilled.
6. User confirms completion.
7. Request is closed.

---

## Difference Between Incident and Service Request

### Incident

Unexpected issue requiring restoration of service.

Example:

Outlook crashes.

---

### Service Request

User requests a standard service.

Example:

Install Microsoft Office.

---

## Best Practices

- Verify approvals before fulfillment.
- Follow standard procedures.
- Update users regularly.
- Close requests only after confirmation.
