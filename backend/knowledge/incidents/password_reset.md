# Password Reset

...

## Purpose

Provide a secure and standardized process for resetting a user's password while maintaining account security.

---

## When to Use

Use this procedure when:

- User forgot password.
- Password has expired.
- User account is locked.
- Temporary password is required.

---

## Prerequisites

Before resetting the password:

- Verify the user's identity.
- Confirm employee ID or username.
- Verify department or manager if required.
- Ensure the request is authorized.

---

## Procedure

### Step 1

Receive the password reset request.

---

### Step 2

Verify the user's identity.

---

### Step 3

Open Active Directory Users and Computers (or the organization's identity management system).

---

### Step 4

Locate the user's account.

---

### Step 5

Select **Reset Password**.

---

### Step 6

Assign a temporary password following the organization's password policy.

---

### Step 7

Enable:

- User must change password at next login.

---

### Step 8

Provide the temporary password securely.

Never send passwords through unsecured channels.

---

### Step 9

Ask the user to log in and confirm successful access.

---

## Validation

Confirm that:

- User can log in successfully.
- Password change is completed.
- No account lock exists.

---

## Escalation

Escalate if:

- User account cannot be located.
- Active Directory is unavailable.
- Multiple password reset attempts fail.
- Identity cannot be verified.

---

## Best Practices

- Never disclose passwords to unauthorized persons.
- Verify identity before every reset.
- Follow password policy.
- Document the activity in the ticket.
